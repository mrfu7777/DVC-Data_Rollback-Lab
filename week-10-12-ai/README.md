# Week 10-12 Green AI Experiment

## Objective

This lab measures the carbon footprint of a custom machine learning experiment with CodeCarbon. The experiment uses a dataset and architecture that are different from the classroom examples, then compares model quality against emitted CO2.

## Dataset

The experiment uses the real UCI Wine Quality red wine dataset, downloaded as a CSV file from the UCI Machine Learning Repository.

- Source: https://archive.ics.uci.edu/ml/datasets/wine+quality
- Local file: `winequality-red.csv`
- Rows: 1,599 red wine samples
- Features: 11 physicochemical wine measurements
- Target: `quality >= 6` is labeled as high quality, and lower scores are labeled as lower quality

## Model Design

Two Random Forest setups are trained and compared:

- `small_random_forest_50_trees`: 50 trees with a maximum depth of 8.
- `larger_random_forest_400_trees`: 400 trees with unlimited depth.

This setup tests whether a larger classical ML architecture improves accuracy enough to justify the extra carbon emissions.

## Files

- `run_green_ai_experiment.py`: reproducible experiment script.
- `winequality-red.csv`: real external dataset used for the experiment.
- `emissions.csv`: CodeCarbon output file generated by the training runs.
- `metrics.json`: structured summary of accuracy, loss, emissions, duration, and carbon ROI.
- `requirements.txt`: Python dependencies used for this experiment.

## How to Run

```bash
python -m venv .venv
.venv/bin/python -m pip install -r week-10-12-ai/requirements.txt
.venv/bin/python week-10-12-ai/run_green_ai_experiment.py
```

## Result Summary

The generated `metrics.json` file contains the final results and a short interpretation of the trade-off between accuracy and CO2 emissions.
